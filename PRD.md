# myroutine PRD

## 1. 제품 개요

**myroutine**은 하루 일정과 남은 자유시간을 기준으로 오늘 할 루틴을 고르고, 루틴 실행에 맞는 YouTube 영상/BGM을 추천해 주는 생산성 웹앱이다.

핵심 아이디어는 **"영상이 끝나면 루틴도 자연스럽게 끝난다"**는 흐름이다. 사용자는 별도 타이머를 설정하지 않아도 영상 길이를 루틴 실행 단위로 활용할 수 있다.

## 2. 문제 정의

루틴을 실천하려는 사용자는 매번 다음 문제를 겪는다.

- 오늘 실제로 시간이 얼마나 남았는지 계산하기 번거롭다.
- 루틴을 시작하기 전에 YouTube 영상이나 BGM을 다시 검색해야 한다.
- 루틴 시간이 애매하면 무엇을 할지 결정하는 데 시간이 든다.
- 타이머 앱, 영상 앱, 메모 앱을 오가며 실행 흐름이 끊긴다.

## 3. 목표

- 하루 시작 시 짧은 질문만으로 오늘의 자유시간을 계산한다.
- 사용자가 자유시간 안에서 할 수 있는 활동을 빠르게 고르게 한다.
- 활동별 실행용 YouTube 영상/BGM을 한 화면에서 선택하게 한다.
- 선택한 시간표와 영상을 저장해 다시 접속했을 때 현재 시간대에 맞는 안내를 보여준다.
- Copilot SDK 기반 AI 채팅으로 취미/활동 설정을 자연스럽게 수집한다.

## 4. 타겟 사용자

- 퇴근 후 남는 시간에 취미나 공부 루틴을 만들고 싶은 개인 사용자
- 스트레칭, 악기 연습, 드로잉, 공부처럼 영상/BGM과 함께 실행하기 좋은 루틴이 있는 사용자
- 루틴 앱은 쓰고 싶지만 복잡한 캘린더 설정은 부담스러운 사용자

## 5. 핵심 사용자 플로우

### 5.1 첫 설정

1. 사용자가 `/setup`에서 AI 채팅을 시작한다.
2. Copilot SDK 기반 채팅이 취미, 하고 싶은 일, 선호 BGM/영상 검색어를 물어본다.
3. 사용자는 직접 입력하거나 태그 버튼을 눌러 빠르게 답한다.
4. 설정된 취미 목록은 `localStorage`에 저장된다.
5. 메인 페이지에서 저장된 취미 목록을 자동 반영한다.

### 5.2 하루 시작

1. 사용자가 메인 화면에서 **하루 시작**을 누른다.
2. 앱이 오늘 수영 여부, 업무 마무리 시각, 재택 여부를 묻는다.
3. 앱이 현재 시각부터 자정까지의 업무/수영/이동/자유시간을 계산한다.
4. 24시간 타임라인과 자유시간 카드로 결과를 보여준다.
5. 사용자가 가능한 활동을 선택한다.
6. 앱이 남은 시간에 맞춰 추가 활동과 시간표 초안을 추천한다.
7. 사용자가 시간표를 확정하면 영상 선택 단계로 이동한다.
8. 사용자가 활동별 YouTube 영상/BGM을 선택한다.
9. 최종 확인 후 오늘의 루틴 기록을 저장한다.

### 5.3 다시 접속

1. 앱이 오늘 저장된 루틴 기록을 확인한다.
2. 현재 시간이 업무 중이면 업무 상태를 보여준다.
3. 현재 시간이 루틴 시간대이면 해당 활동과 선택한 영상을 보여준다.
4. 루틴 시간이 지났으면 다음 활동 또는 오늘의 진행 상태를 보여준다.

## 6. 주요 기능 요구사항

| 구분 | 요구사항 |
| --- | --- |
| 브랜딩 | 앱 이름은 `myroutine`으로 표시하고, 전용 파비콘을 사용한다. |
| 온보딩 설명 | 메인 화면 오른쪽 상단의 `i` 아이콘을 누르면 앱 작동 방식 팝업을 보여준다. |
| 하루 시작 질문 | 수영 여부, 업무 종료 시각, 재택 여부를 입력받는다. |
| 자유시간 계산 | 현재 시각, 업무 종료, 수영 종료, 출근/퇴근 준비 시간을 반영해 자유시간을 계산한다. |
| 타임라인 | 하루를 업무, 수영, 이동/정리, 자유시간 구간으로 시각화한다. |
| 활동 추천 | 사용 가능한 시간보다 긴 활동은 추천에서 제외한다. |
| 시간표 초안 | 선택한 활동과 남은 시간에 맞는 추가 활동을 제안한다. |
| 영상 선택 | 영상 지원 활동은 YouTube 검색 결과에서 실행용 영상/BGM을 선택하게 한다. |
| 오늘 기록 | 확정한 시간표와 영상 선택을 `localStorage`에 저장한다. |
| 설정 채팅 | Copilot SDK 스트리밍 채팅으로 사용자 취미 설정을 수집한다. |

## 8. 비기능 요구사항

- 모바일 화면에서도 하루 시작 플로우와 영상 선택이 끊기지 않아야 한다.
- AI 응답은 스트리밍으로 표시해 대기감을 줄인다.
- YouTube API 키는 클라이언트에 노출하지 않고 서버 API route에서 사용한다.
- 사용자의 루틴 데이터는 로그인 없이 브라우저 `localStorage`에 저장한다.
- 최종 저장 전 사용자가 시간표와 영상 선택을 확인할 수 있어야 한다.
- 영상 검색 실패 시 사용자가 루틴 자체를 진행할 수 있도록 오류를 명확히 보여준다.

## 10. 성공 기준

- 사용자가 1분 안에 오늘의 자유시간을 확인하고 첫 활동을 고를 수 있다.
- 사용자가 루틴 실행 전 YouTube를 직접 검색하지 않아도 된다.
- 저장된 오늘 루틴을 다시 접속했을 때 현재 시간대에 맞게 확인할 수 있다.
- 설정 페이지에서 AI 채팅으로 취미 목록을 추가/수정할 수 있다.
